虽然没有了got表，但是我们可以通过bss段的stdin进行leak。要做的操作有：

  • 控制write参数将stdin地址leak
  • 控制read参数读取one_gadget地址并将其写入到已知地址
  • 控制程序执行流，到one_gadget

通过write进行leak后，如果想要再次控制rax那么就会进行leave ret，进行了栈迁移。这是我们无法控制的，并且我们没有在fake stack中布置栈结构。所以我们先进行read，提前布置好栈的结构。然后调用write进行leak。之后因为还要接受one_gadget的地址，并将其写入可控地址，同时我们要进行栈的迁移。所以回到0x40125f的位置正好帮助我们完成了这个操作。

那么布置的栈结构是什么样的呢？当程序执行到fake stack中时，已经完成了，leak与栈迁移，rax也被置为0，这时我们只要调用read，并将one_gadget写到read返回之后即可。

_DWORD *__cdecl eval(_DWORD *a1, char a2)
{
  _DWORD *result; // eax

  if ( a2 == '+' )
  {
    a1[*a1 - 1] += a1[*a1];
  }
  else if ( a2 > '+' )
  {
    if ( a2 == '-' )
    {
      a1[*a1 - 1] -= a1[*a1];
    }
    else if ( a2 == '/' )
    {
      a1[*a1 - 1] /= (int)a1[*a1];
    }
  }
  else if ( a2 == '*' )
  {
    a1[*a1 - 1] *= a1[*a1];
  }
  result = a1;
  --*a1;
  return result;
}
